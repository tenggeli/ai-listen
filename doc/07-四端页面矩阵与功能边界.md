# listen 四端页面矩阵与功能边界（精简版）

## 1. 四端定位
- 用户 App：高频消费与深交互主阵地
- 用户 Web：轻访问、传播和低门槛入口
- 管理 App：移动审批与应急处理
- 管理 Web：完整运营后台（复杂查询、配置、RBAC）

## 2. 当前代码落位（2026-03-26）

| 端 | 当前页面 |
| --- | --- |
| 用户 App | `首页`、`登录`、`我的资料` |
| 管理 App | `控制台首页`、`登录` |
| 用户 Web | `/`（首页）、`/login`（短信登录）、`/orders`（我的订单）、`/profile`（个人中心） |
| 管理 Web | `/`（控制台占位） |

## 3. 页面归属规则
1. 高频交互与设备能力相关功能优先放 App。
2. 传播触达与轻流程功能优先放用户 Web。
3. 复杂表格、批量操作、配置与权限管理只放管理 Web。
4. 管理 App 只保留高时效待办，不承载全量后台能力。

## 4. 功能边界
- 用户侧（App/Web）共用 `/api/v1`，禁止自定义状态语义分叉。
- 管理侧（App/Web）共用 `/api/v1/admin` 与同一套 RBAC 权限点。
- 页面可以裁剪，业务动作语义不能裁剪。
