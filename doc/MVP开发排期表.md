# ai-listen MVP 开发排期表

## 1. 文档说明
本排期表用于 ai-listen 项目 MVP 阶段的产品、设计、研发、测试协同。

目标：
- 先完成最小可运行闭环
- 验证 AI 匹配 + 下单 + 履约 + 结算 + 评价
- 为后续广场、声音、VIP 等模块打基础

---

## 2. MVP 目标范围
MVP 必须完成：
- 用户登录注册
- 服务提供方入驻审核
- 首页 AI 输入与推荐
- 服务方详情
- 下单与支付
- 接单、出发、到达、开始、完单
- 订单详情与状态流转
- 双方评价
- 后台审核与订单查看

MVP 暂不重点做：
- 广场完整社区生态
- 声音完整内容生态
- 红娘/管家
- 高级私聊系统
- 复杂营销活动

---

## 3. 团队角色建议
- 产品经理：1
- UI/UX：1
- 前端开发（APP）：1-2
- 后端开发（Go）：2
- 测试：1
- 运维/部署：1（可兼职）

---

## 4. 周期建议
建议 MVP 周期：**8 周**

## 5. 详细排期

- 明确 MVP 范围

- 明确设计风格板

- 输出测试计划
- 梳理验收点

## 8. 风险点与提醒
- 首页动效不要过早投入过重研发，先做可用版本
- AI 前期先用规则/标签推荐，后续再智能升级
- 订单状态机必须优先设计清楚
- 支付与结算逻辑必须提前联调
- 风控和实名认证不能拖到最后

---

## 9. 建议交付顺序
1. 先交互原型
2. 再 UI 视觉稿
3. 再数据库与 API 定稿
4. 再研发并行开发
5. 再联调测试

---

## 10. 结论
MVP 的目标不是把所有功能做满，而是优先验证最关键的商业闭环：
**用户愿不愿意通过 AI 发需求 -> 是否能顺利匹配 -> 是否能顺利下单 -> 服务方能否稳定履约 -> 平台能否安全结算。**
