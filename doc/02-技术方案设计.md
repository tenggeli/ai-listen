# listen 技术方案设计（精简版）

## 1. 技术栈
- 后端：Go、Gin、Viper、Zap
- 存储：MySQL + 内存双实现（`MYSQL_DSN` 为空时使用内存）
- Web：Vue 3 + Vite（用户端、管理端）
- App：uni-app + Vue 3（用户端、管理端）
- 部署：Nginx、Docker Compose（示例）

## 2. 架构总览
- API 入口：`/api/v1`
- 管理 API：`/api/v1/admin`
- 鉴权方式：`Authorization: Bearer <token>`
- 后端结构：`router -> control -> model(store)`
- App 共享层：`app/packages/shared`（HTTP、API 类型、设计令牌）

## 3. 目录现状
- `backend/cmd/server`：启动入口
- `backend/internal/router`：路由注册
- `backend/internal/control`：按域组织控制器
- `backend/internal/model`：领域模型 + MySQL/内存实现
- `backend/migrations`：建表 SQL
- `web/apps/user-web`：用户 Web（当前 1 个首页路由）
- `web/apps/admin-web`：管理 Web（当前 1 个控制台路由）
- `app/apps/user-app`：用户 App（首页/登录/资料）
- `app/apps/admin-app`：管理 App（首页/登录）

## 4. 后端模块（已注册）
`health`、`auth`、`user`、`me`、`serviceitem`、`provider`、`providercenter`、`order`、`payment`、`review`、`complaint`、`post`、`audio`、`aimatch`、`admin`

## 5. 关键实现约定
- 轻模块：`routes.go + controller.go (+ dto.go)`
- 复杂模块：`routes.go + *_controller.go (+ dto.go)`
- 存储抽象：统一 `model.Store` 接口，便于内存/MySQL 切换
- 管理端权限：`AdminRequired + RequirePermission(...)`

## 6. 已知缺口
- 内容域已有基础落库能力，但 `audio`、`aimatch` 等模块仍偏占位
- 复杂编排尚未下沉到 `service` 层
- Web/App 页面仍以基础骨架为主
- 退款、结算、提现等资金链路未闭环，且当前开发优先级最低
