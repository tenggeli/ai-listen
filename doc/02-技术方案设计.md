# listen 技术方案

## 1. 技术栈
- 后端：Go、Gin、Viper、Zap、`database/sql`
- 存储：MySQL，未配置时可回退到内存存储
- 前端：Vue 3 + Vite
- 部署：Nginx、Docker Compose 示例

## 2. 当前结构
- `backend/cmd/server`：启动入口
- `backend/internal/app`：配置、日志、存储初始化
- `backend/internal/router`：路由注册
- `backend/internal/modules/*`：业务模块
- `backend/internal/store`：存储抽象、内存实现、MySQL 实现
- `backend/migrations`：初始化 SQL

## 3. 开发架构约束
- 后端模块默认遵循面向对象分层：`handler -> service -> store`
- 业务规则优先沉淀在 Service 或 Domain Object，不直接堆在 Handler 中
- 数据访问通过接口抽象隔离具体实现，便于内存版与 MySQL 版切换
- 详细规范见《开发规范与面向对象设计》

## 4. 当前后端模块
- 已接通核心链路：
  - `auth`
  - `provider-center`
  - `order`
  - `payment`
  - `admin` 中的服务方审核
- 已注册但多为占位：
  - `aimatch`
  - `review`
  - `complaint`
  - `post`
  - `audio`
  - 其他后台管理接口

## 5. 存储策略
- 默认：内存存储，适合本地联调与测试
- MySQL：配置 `MYSQL_DSN` 后启用
- 启动时会自动执行初始化建表 SQL，并写入基础服务项目数据

## 6. 订单状态机
- 待支付
- 待接单
- 已接单
- 已出发
- 已到达
- 服务中
- 待确认完成
- 已完成
- 已取消
