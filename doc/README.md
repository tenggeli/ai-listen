# Listen 文档索引

本目录已按“原始需求输入”和“面向 AI Coding 的执行输入”分层整理，目标是让 Codex 在进入项目时能快速找到正确上下文，而不是在多份长文档之间反复猜测。

## 推荐阅读顺序

如需让 Codex 参与开发，建议按下面顺序读取：

1. `doc/03-AI开发/00-Codex-启动入口.md`
2. `doc/03-AI开发/01-技术方案设计.md`
3. `doc/03-AI开发/02-AI-Coding-指导.md`
4. `doc/01-产品设计/01-产品设计总览.md`
5. `doc/02-PRD/用户端PRD.md`
6. `doc/02-PRD/服务方端PRD.md`
7. `doc/05-管理后台原型/` 下的双后台原型
8. `doc/04-原型素材/` 下的用户端页面原型与素材

## 文档优先级

当不同文档出现描述差异时，按以下优先级裁决：

1. `doc/03-AI开发/00-Codex-启动入口.md`
2. `doc/03-AI开发/01-技术方案设计.md`
3. `doc/03-AI开发/02-AI-Coding-指导.md`
4. `doc/01-产品设计/01-产品设计总览.md`
5. `doc/02-PRD/`
6. `doc/01-产品设计/00-产品构想原稿.md`
7. `doc/05-管理后台原型/`
8. `doc/04-原型素材/`

说明：

- `03-AI开发` 目录是专门为 AI Coding 准备的执行输入，优先级最高。
- `01-产品设计/00-产品构想原稿.md` 保留原始想法，适合补充理解，不适合直接当最终实现依据。
- `05-管理后台原型` 用于定义双后台的页面拆分与职责边界。
- 原型 HTML 主要用于视觉和页面结构参考，不直接作为业务规则来源。

## 目录说明

### `01-产品设计`

- `00-产品构想原稿.md`：最初的产品发散输入，保留原始想法。
- `01-产品设计总览.md`：整合后的产品定位、信息架构与核心页面设计。

### `02-PRD`

- `用户端PRD.md`：用户端详细需求。
- `服务方端PRD.md`：服务方详细需求。

### `03-AI开发`

- `00-Codex-启动入口.md`：Codex 首读文档，说明当前阶段应该先做什么。
- `01-技术方案设计.md`：面向对象建模、系统边界、模块拆分、领域模型。
- `02-AI-Coding-指导.md`：AI 写代码时的目录、命名、任务粒度和开发顺序约束。

### `05-管理后台原型`

- `listen_admin_prototype_hub.html`：双后台原型总览。
- `listen_provider_console.html`：服务方管理后台原型。
- `listen_platform_admin_console.html`：平台管理后台原型。

### `04-原型素材`

- `listen_home_v2_dark.html`：首页原型。
- `listen_page_chat.html`：AI 对话页原型。
- `listen_page_sound.html`：声音页原型。
- `logo.jpg`：视觉素材。

## 给 Codex 的使用建议

- 先按推荐顺序读取，不要直接从 PRD 开始写代码。
- 新功能实现时，先确认是否属于 P0/P1，避免先做支付、结算、提现这类低优先模块。
- 若任务只给一句模糊需求，优先参考 `03-AI开发` 目录，而不是自行扩散范围。
- 若是用户端页面开发，再结合 `04-原型素材` 中的 HTML 结构做视觉还原。
- 若是后台页面开发，优先参考 `05-管理后台原型` 的双后台拆分，不再按单一管理后台理解。
