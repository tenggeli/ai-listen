# listen 数据库设计

## 1. 选型
- MySQL 8.0+
- `utf8mb4`
- 金额单位统一为分
- 关键业务保留状态日志

## 2. 当前核心表

### 认证
- `users`
- `user_auths`
- `auth_sms_codes`
- `auth_tokens`

### 服务提供方
- `providers`
- `provider_profiles`
- `provider_service_items`
- `audit_records`

### 交易
- `service_items`
- `orders`
- `order_status_logs`
- `payments`

### 后台 RBAC
- `admin_users`
- `admin_roles`
- `admin_permissions`
- `admin_user_roles`
- `admin_role_permissions`

## 3. 关键状态

### providers.audit_status
- `1` 待审核
- `2` 通过
- `3` 拒绝

### orders.status
- `10` 待支付
- `20` 待接单
- `30` 已接单
- `40` 已出发
- `50` 已到达
- `60` 服务中
- `70` 待确认完成
- `80` 已完成
- `90` 已取消

## 4. 当前实现说明
- 初始化 SQL 位于 `backend/migrations/000001_init_schema.sql`
- 启动应用并配置 `MYSQL_DSN` 后会自动建表
- 当前 MySQL 已覆盖认证、服务方入驻审核、订单主链路、后台 RBAC 权限持久化与角色分配
