# listen 数据库设计（精简版）

> 以 `backend/migrations/000001_init_schema.sql` 为准。

## 1. 基础约定
- MySQL 8.0+
- 字符集：`utf8mb4`
- 金额统一使用“分”（`BIGINT`）
- 状态型业务保留状态/操作日志表

## 3. 关键状态

### 3.1 服务方审核状态（`providers.audit_status`）
- `0` 未申请/默认
- `1` 待审核
- `2` 审核通过
- `3` 审核拒绝

### 3.2 服务方工作状态（`providers.work_status`）
- `1` 休息/不可接单
- `2` 可接单

### 3.3 订单状态（`orders.status`）
- `10` 待支付
- `20` 待接单
- `30` 已接单
- `40` 已出发
- `50` 已到达
- `60` 服务中
- `70` 待确认完成
- `80` 已完成
- `90` 已取消

### 3.4 支付状态（`payments.status`）
- `10` 待支付
- `20` 已支付
- `30` 支付失败

## 4. 当前实现关注点
- `payment_callback_idempotency` 用于按 `channel + notifyId` 做支付回调幂等保护
- 内容域表已被 `post/review/complaint` 基础链路实际使用，不再只是预留表

## 5. 运行说明
- 未配置 `MYSQL_DSN`：走内存存储
- 配置 `MYSQL_DSN`：启动时自动建表并初始化种子数据
