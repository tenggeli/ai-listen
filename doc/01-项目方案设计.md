# listen 项目方案

## 1. 定位
- 以 AI 指令式交互为入口的陪伴服务平台
- 连接用户、服务提供方、平台后台三类角色
- 核心闭环是下单、履约、结算、评价与风控

## 2. 当前范围
- 用户端：登录、浏览服务方、下单、支付、查看订单
- 服务方端：入驻申请、资料配置、服务项目配置、接单履约
- 后台：审核服务方、查看核心业务数据

## 3. 当前已落地
- 短信登录与 token 刷新
- 服务提供方入驻申请与后台审核
- 服务项目配置
- 订单主链路：
  - 创建订单
  - 创建支付单
  - 接单
  - 出发
  - 到达
  - 开始服务
  - 完单
  - 用户确认完成

## 4. 当前未落地
- AI 推荐真实编排
- 评价、投诉、退款、结算、提现
- 广场、声音内容完整能力
- 后台管理员登录与权限体系

## 5. 后续优先级
1. 支付回调、退款、结算、提现审核
2. 后台登录鉴权与核心管理列表
3. 订单超时、异常单、审计查询
4. 评价、投诉、内容与 AI 推荐
