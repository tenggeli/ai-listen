服务方后台 + 平台管理后台 双后台原型总览

本轮原型基于 doc/02-PRD/服务方端PRD.md 拆出两套后台。服务方后台聚焦入驻、履约、收益与经营；平台管理后台聚焦审核、风控、订单调度、资金与平台配置，形成平台供给侧完整管理闭环。

后台数量
2 套
核心页面
15+
原型风格
后台线框高保真

拆分原则

服务方后台覆盖服务方从入驻审核、资料维护、接单履约、动态经营到收益提现的日常操作面板。
平台管理后台覆盖服务方审核、订单监管、风控处罚、内容审核、结算提现、基础配置与经营总览。
画布建议后续进入 Figma 时可直接按“首页总览 + 列表页 + 详情抽屉 + 处理弹窗”的分层方式建 Frame。

后台原型入口

Console A

服务方后台原型

聚焦服务方日常运营效率，突出状态切换、待处理订单、服务项目、信用评分、收益结算和内容经营等高频任务。

工作台 订单履约 收益提现 资料管理
Console B

平台管理后台原型

面向运营、审核、客服与财务角色，统一处理服务方准入、订单异常、投诉仲裁、资金放款、规则配置与经营概览。

审核中心 风控中心 资金结算 平台配置

页面清单

服务方后台

  • 工作台总览
  • 入驻申请 / 审核状态
  • 资料管理
  • 服务项目与价格
  • 订单列表 / 订单详情 / 履约操作
  • 主动邀约 / 用户线索池
  • 收益结算 / 提现
  • 信用评分 / 评价 / 圈子动态

平台管理后台

  • 经营总览看板
  • 服务方审核中心
  • 订单监管与状态流转
  • 投诉仲裁 / 风控预警
  • 内容审核中心
  • 资金结算 / 提现审核
  • 积分信用规则
  • 服务类目 / 价格 / 推送配置
