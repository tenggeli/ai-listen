# listen 接口设计规范

## 1. 文档目标
- 统一 listen 项目的接口风格、路径规划、鉴权方式、错误码和模块边界。
- 区分“当前已实现接口”和“规划中接口”，降低前后端联调偏差。

## 2. 总体约定

### 2.1 路径规范
- 用户端统一前缀：`/api/v1`
- 管理后台统一前缀：`/api/v1/admin`

### 2.2 请求头约定
- `Content-Type: application/json`
- `Authorization: Bearer <token>`
- `X-Request-Id: <uuid>` 可选

### 2.3 响应结构

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "traceId": "trace-id"
}
```

## 3. 鉴权说明

### 3.1 当前实现
- 登录成功后返回 `accessToken` 与 `refreshToken`
- 当前代码采用 Bearer Token + 服务端 token 表校验
- 服务提供方接口本质上仍使用用户身份，再额外校验服务方状态

### 3.2 后续规划
- 管理后台补独立管理员登录
- 增加 RBAC
- 可视需要演进为 JWT 或混合方案

## 4. 错误码规范

### 4.1 通用错误码
| 错误码 | 说明 |
|---|---|
| 0 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录或 token 无效 |
| 40300 | 无权限访问 |
| 40400 | 资源不存在 |
| 40900 | 状态冲突 |
| 50000 | 系统异常 |

### 4.2 当前常用业务错误
| 错误码 | 说明 |
|---|---|
| 100101 | 验证码错误或已过期 |
| 100201 | 服务方审核未通过 |
| 100301 | 订单状态不允许当前操作 |
| 100302 | 订单不存在 |

## 6. 当前真正接通业务存储的接口
- 认证：短信发送、短信登录、刷新 token
- 服务提供方：申请入驻、查看申请状态、更新资料、更新服务项目、切换工作状态
- 后台：服务提供方审核通过、审核拒绝
- 订单：创建订单、创建支付单、接单、出发、到达、开始服务、完单、确认完成、查看状态日志

## 7. 当前占位接口说明
- 微信真实登录换取 openid
- 后台管理员真实登录与权限校验
- 退款、结算、提现、评价、投诉
- 广场、声音、收藏、点赞等内容链路的大部分深层逻辑
- AI 推荐真实编排与排序

## 8. 设计要求
- 创建订单、支付、退款、提现等写接口后续必须支持幂等
- 涉及金额变化的接口必须记录操作日志
- 支付回调必须进行签名校验
- 时间字段统一返回 `YYYY-MM-DD HH:mm:ss`
- 金额字段优先使用“分”为单位

## 9. 当前联调建议
- 前后端联调时，优先使用已接通业务存储的接口
- 对于占位接口，应在联调文档或 Mock 中明确说明“仅回显，不落库”
- 后续建议补一份 OpenAPI 或 Apifox 集合，避免接口细节再次偏离代码实现
