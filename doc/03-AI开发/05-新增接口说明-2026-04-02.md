# Listen 新增接口说明（OpenAPI 风格最小版）

更新时间：2026-04-03

## 用户侧：设置持久化

### `GET /api/v1/users/me/settings`

- 鉴权：`Authorization: Bearer <access_token>`
- 请求参数：无
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "preference": {
      "prefer_same_city_providers": true,
      "auto_play_sound_preview": true,
      "hide_offline_providers": false
    },
    "notification": {
      "order_status_update": true,
      "complaint_result_notice": true,
      "marketing_activity": false
    },
    "privacy": {
      "profile_public_visible": true,
      "personalized_recommendation": true,
      "risk_control_data_sharing": true
    }
  },
  "request_id": "req_xxx"
}
```

- 失败响应：
1. `401 unauthorized`：未登录或 token 无效
2. `404 user not found`：用户不存在
3. `501 settings persistence only available in mysql mode`：当前为 memory 模式

### `PUT /api/v1/users/me/settings`

- 鉴权：`Authorization: Bearer <access_token>`
- 请求体：

```json
{
  "preference": {
    "prefer_same_city_providers": true,
    "auto_play_sound_preview": false,
    "hide_offline_providers": true
  },
  "notification": {
    "order_status_update": true,
    "complaint_result_notice": true,
    "marketing_activity": false
  },
  "privacy": {
    "profile_public_visible": true,
    "personalized_recommendation": true,
    "risk_control_data_sharing": true
  }
}
```

- 成功响应：`200`，响应体同 `GET /users/me/settings`
- 失败响应：
1. `400 invalid request body|invalid input`
2. `401 unauthorized`
3. `404 user not found`
4. `501 settings persistence only available in mysql mode`

## 平台管理侧：服务项目管理（首版）

### `GET /api/v1/admin/service-items`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- Query 参数：
1. `provider_id`（可选）
2. `category_id`（可选）
3. `status`（可选，`active|inactive`）
4. `keyword`（可选，匹配标题/描述/服务方名）
5. `page`（可选，默认 1）
6. `page_size`（可选，默认 10，最大 50）
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": "si_001",
        "provider_id": "p_pub_001",
        "provider_name": "暖心倾听师 · 小林",
        "category_id": "cat_chat",
        "title": "线上倾听 30 分钟",
        "description": "适合情绪安抚、想找人说话、睡前放松。",
        "price_amount": 99,
        "price_unit": "30分钟",
        "support_online": true,
        "sort_order": 1,
        "service_status": "active"
      }
    ],
    "total": 1
  },
  "request_id": "req_xxx"
}
```

- 失败响应：
1. `400 invalid input`（状态枚举不合法等）
2. `401 unauthorized`
3. `501 service item management only available in mysql mode`

### `GET /api/v1/admin/service-items/{id}`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 成功响应：`200`，`data` 字段结构同列表单项
- 失败响应：
1. `400 invalid service item id|invalid input`
2. `401 unauthorized`
3. `404 service item not found`
4. `501 service item management only available in mysql mode`

### `POST /api/v1/admin/service-items/{id}/activate`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：无
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "si_001",
    "service_status": "active"
  },
  "request_id": "req_xxx"
}
```

- 失败响应：`400|401|404|501`（同上）

### `POST /api/v1/admin/service-items/{id}/deactivate`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：无
- 成功响应：`200`，`service_status` 为 `inactive`
- 失败响应：`400|401|404|501`（同上）

## 平台管理侧：订单与投诉管理（首版）

### `GET /api/v1/admin/orders`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- Query 参数：
1. `status`（可选）
2. `keyword`（可选，匹配订单号/用户/服务方/服务项目）
3. `page`（可选，默认 1）
4. `page_size`（可选，默认 10，最大 50）
- 成功响应：`200`，`data.items` 为订单摘要列表，`data.total` 为总数
- 失败响应：`400|401|500`

### `GET /api/v1/admin/orders/{id}`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 成功响应：`200`，返回 `order`、可选 `feedback`、`action_logs`
- 失败响应：
1. `400 invalid input`
2. `401 unauthorized`
3. `404 order not found`

### `POST /api/v1/admin/orders/{id}/intervene`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：

```json
{
  "reason": "人工介入"
}
```

- 成功响应：`200`，返回更新后的 `order` 与 `audit`
- 失败响应：`400|401|404|409`

### `POST /api/v1/admin/orders/{id}/close`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：同 `intervene`
- 成功响应：`200`，返回更新后的 `order` 与 `audit`
- 失败响应：`400|401|404|409`

### `GET /api/v1/admin/complaints`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- Query 参数：
1. `page`（可选，默认 1）
2. `page_size`（可选，默认 10，最大 50）
- 成功响应：`200`，返回投诉列表（含 `complaint_status`、`order`、`feedback`）及 `total`
- 失败响应：`400|401|500`

### `GET /api/v1/admin/complaints/{id}`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 成功响应：`200`，返回投诉详情（`complaint_status`、`order`、`feedback`、`action_logs`）
- 失败响应：`400|401|404`

### `POST /api/v1/admin/complaints/{id}/intervene`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：

```json
{
  "reason": "平台人工介入"
}
```

- 成功响应：`200`，返回更新后的 `complaint` 与 `audit`
- 失败响应：`400|401|404|409`

### `POST /api/v1/admin/complaints/{id}/resolve`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：同 `intervene`
- 成功响应：`200`，返回更新后的 `complaint` 与 `audit`
- 失败响应：`400|401|404|409`

## 服务方侧：鉴权与订单履约（后端首版）

### `POST /api/v1/provider/auth/login/mock`

- 鉴权：无
- 请求体：

```json
{
  "account": "provider",
  "password": "provider123"
}
```

- 成功响应：`200`，返回 `access_token`、`provider_id`
- 失败响应：`400|401|404`

### `GET /api/v1/provider/profile`

- 鉴权：`Authorization: Bearer <provider_access_token>`
- 成功响应：`200`，返回服务方账号信息（`provider_id`、`account`、`display_name`、`status`、`city_code`）
- 失败响应：`401|404`

### `GET /api/v1/provider/orders`

- 鉴权：`Authorization: Bearer <provider_access_token>`
- Query 参数：`page`、`page_size`（可选）
- 成功响应：`200`，返回服务方订单列表及 `total`
- 失败响应：`400|401|403|404`

### `GET /api/v1/provider/orders/{id}`

- 鉴权：`Authorization: Bearer <provider_access_token>`
- 成功响应：`200`，返回订单详情
- 失败响应：`400|401|403|404`

### `POST /api/v1/provider/orders/{id}/accept|depart|arrive|start|complete`

- 鉴权：`Authorization: Bearer <provider_access_token>`
- 请求体：无
- 成功响应：`200`，返回 `{ id, status }`
- 失败响应：`400|401|403|404|409`

## 联调 `curl` 清单（最小可执行）

默认后端地址：`http://127.0.0.1:8080`

### 1) 用户登录并获取 token

```bash
curl -s -X POST 'http://127.0.0.1:8080/api/v1/auth/login/sms' \
  -H 'Content-Type: application/json' \
  -d '{"phone":"13800113800","verify_code":"123","agreement_accepted":true}'
```

从返回中取 `data.access_token` 保存为 `USER_TOKEN`。

### 2) 读取用户设置

```bash
curl -s 'http://127.0.0.1:8080/api/v1/users/me/settings' \
  -H "Authorization: Bearer ${USER_TOKEN}"
```

### 3) 保存用户设置

```bash
curl -s -X PUT 'http://127.0.0.1:8080/api/v1/users/me/settings' \
  -H "Authorization: Bearer ${USER_TOKEN}" \
  -H 'Content-Type: application/json' \
  -d '{
    "preference":{"prefer_same_city_providers":true,"auto_play_sound_preview":false,"hide_offline_providers":true},
    "notification":{"order_status_update":true,"complaint_result_notice":true,"marketing_activity":false},
    "privacy":{"profile_public_visible":true,"personalized_recommendation":true,"risk_control_data_sharing":true}
  }'
```

### 4) 管理员登录并获取 token

```bash
curl -s -X POST 'http://127.0.0.1:8080/api/v1/admin/auth/login/mock' \
  -H 'Content-Type: application/json' \
  -d '{"account":"admin","password":"admin123"}'
```

从返回中取 `data.access_token` 保存为 `ADMIN_TOKEN`。

### 5) 查询服务项目列表

```bash
curl -s 'http://127.0.0.1:8080/api/v1/admin/service-items?status=active&page=1&page_size=10' \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

从返回中取一个 `id`（如 `si_001`）保存为 `ITEM_ID`。

### 6) 查看服务项目详情

```bash
curl -s "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 7) 停用/启用服务项目

```bash
curl -s -X POST "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}/deactivate" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

curl -s -X POST "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}/activate" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 8) 查询后台订单与投诉（平台管理侧）

```bash
curl -s 'http://127.0.0.1:8080/api/v1/admin/orders?page=1&page_size=10' \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

curl -s 'http://127.0.0.1:8080/api/v1/admin/complaints?page=1&page_size=10' \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 9) 服务方登录并执行履约动作（服务方侧）

```bash
curl -s -X POST 'http://127.0.0.1:8080/api/v1/provider/auth/login/mock' \
  -H 'Content-Type: application/json' \
  -d '{"account":"provider","password":"provider123"}'
```

从返回中取 `data.access_token` 保存为 `PROVIDER_TOKEN`，并执行：

```bash
curl -s 'http://127.0.0.1:8080/api/v1/provider/profile' \
  -H "Authorization: Bearer ${PROVIDER_TOKEN}"

curl -s 'http://127.0.0.1:8080/api/v1/provider/orders?page=1&page_size=10' \
  -H "Authorization: Bearer ${PROVIDER_TOKEN}"
```
