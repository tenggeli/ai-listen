# Listen 新增接口说明（OpenAPI 风格最小版）

更新时间：2026-04-02

## 用户侧：设置持久化

### `GET /api/v1/users/me/settings`

- 鉴权：`Authorization: Bearer <access_token>`
- 请求参数：无
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "preference": {
      "prefer_same_city_providers": true,
      "auto_play_sound_preview": true,
      "hide_offline_providers": false
    },
    "notification": {
      "order_status_update": true,
      "complaint_result_notice": true,
      "marketing_activity": false
    },
    "privacy": {
      "profile_public_visible": true,
      "personalized_recommendation": true,
      "risk_control_data_sharing": true
    }
  },
  "request_id": "req_xxx"
}
```

- 失败响应：
1. `401 unauthorized`：未登录或 token 无效
2. `404 user not found`：用户不存在
3. `501 settings persistence only available in mysql mode`：当前为 memory 模式

### `PUT /api/v1/users/me/settings`

- 鉴权：`Authorization: Bearer <access_token>`
- 请求体：

```json
{
  "preference": {
    "prefer_same_city_providers": true,
    "auto_play_sound_preview": false,
    "hide_offline_providers": true
  },
  "notification": {
    "order_status_update": true,
    "complaint_result_notice": true,
    "marketing_activity": false
  },
  "privacy": {
    "profile_public_visible": true,
    "personalized_recommendation": true,
    "risk_control_data_sharing": true
  }
}
```

- 成功响应：`200`，响应体同 `GET /users/me/settings`
- 失败响应：
1. `400 invalid request body|invalid input`
2. `401 unauthorized`
3. `404 user not found`
4. `501 settings persistence only available in mysql mode`

## 平台管理侧：服务项目管理（首版）

### `GET /api/v1/admin/service-items`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- Query 参数：
1. `provider_id`（可选）
2. `category_id`（可选）
3. `status`（可选，`active|inactive`）
4. `keyword`（可选，匹配标题/描述/服务方名）
5. `page`（可选，默认 1）
6. `page_size`（可选，默认 10，最大 50）
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": "si_001",
        "provider_id": "p_pub_001",
        "provider_name": "暖心倾听师 · 小林",
        "category_id": "cat_chat",
        "title": "线上倾听 30 分钟",
        "description": "适合情绪安抚、想找人说话、睡前放松。",
        "price_amount": 99,
        "price_unit": "30分钟",
        "support_online": true,
        "sort_order": 1,
        "service_status": "active"
      }
    ],
    "total": 1
  },
  "request_id": "req_xxx"
}
```

- 失败响应：
1. `400 invalid input`（状态枚举不合法等）
2. `401 unauthorized`
3. `501 service item management only available in mysql mode`

### `GET /api/v1/admin/service-items/{id}`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 成功响应：`200`，`data` 字段结构同列表单项
- 失败响应：
1. `400 invalid service item id|invalid input`
2. `401 unauthorized`
3. `404 service item not found`
4. `501 service item management only available in mysql mode`

### `POST /api/v1/admin/service-items/{id}/activate`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：无
- 成功响应（`200`）：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "si_001",
    "service_status": "active"
  },
  "request_id": "req_xxx"
}
```

- 失败响应：`400|401|404|501`（同上）

### `POST /api/v1/admin/service-items/{id}/deactivate`

- 鉴权：`Authorization: Bearer <admin_access_token>`
- 请求体：无
- 成功响应：`200`，`service_status` 为 `inactive`
- 失败响应：`400|401|404|501`（同上）

## 联调 `curl` 清单（最小可执行）

默认后端地址：`http://127.0.0.1:8080`

### 1) 用户登录并获取 token

```bash
curl -s -X POST 'http://127.0.0.1:8080/api/v1/auth/login/sms' \
  -H 'Content-Type: application/json' \
  -d '{"phone":"13800113800","verify_code":"123","agreement_accepted":true}'
```

从返回中取 `data.access_token` 保存为 `USER_TOKEN`。

### 2) 读取用户设置

```bash
curl -s 'http://127.0.0.1:8080/api/v1/users/me/settings' \
  -H "Authorization: Bearer ${USER_TOKEN}"
```

### 3) 保存用户设置

```bash
curl -s -X PUT 'http://127.0.0.1:8080/api/v1/users/me/settings' \
  -H "Authorization: Bearer ${USER_TOKEN}" \
  -H 'Content-Type: application/json' \
  -d '{
    "preference":{"prefer_same_city_providers":true,"auto_play_sound_preview":false,"hide_offline_providers":true},
    "notification":{"order_status_update":true,"complaint_result_notice":true,"marketing_activity":false},
    "privacy":{"profile_public_visible":true,"personalized_recommendation":true,"risk_control_data_sharing":true}
  }'
```

### 4) 管理员登录并获取 token

```bash
curl -s -X POST 'http://127.0.0.1:8080/api/v1/admin/auth/login/mock' \
  -H 'Content-Type: application/json' \
  -d '{"account":"admin","password":"admin123"}'
```

从返回中取 `data.access_token` 保存为 `ADMIN_TOKEN`。

### 5) 查询服务项目列表

```bash
curl -s 'http://127.0.0.1:8080/api/v1/admin/service-items?status=active&page=1&page_size=10' \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

从返回中取一个 `id`（如 `si_001`）保存为 `ITEM_ID`。

### 6) 查看服务项目详情

```bash
curl -s "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 7) 停用/启用服务项目

```bash
curl -s -X POST "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}/deactivate" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

curl -s -X POST "http://127.0.0.1:8080/api/v1/admin/service-items/${ITEM_ID}/activate" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```
