# Listen 后续 AI 开发任务列表

本文件用于整理 Listen 项目后续可持续交给 Codex 执行的开发任务。

任务设计原则：

- 一条任务只覆盖一个清晰模块、页面或状态机
- 优先保证 P0 最小闭环，再推进 P1，再考虑 P2
- 默认遵循 `doc/03-AI开发/00-Codex-启动入口.md`
- 默认不让支付、结算、提现阻塞核心主链路

## 1. 优先级说明

### P0

必须优先完成，直接决定项目是否具备最小可运行价值。

### P1

在 P0 核心链路完成后推进，用于补齐交易和运营骨架。

### P2

中后期增强项，不应影响当前 MVP 推进。

## 2. 当前推荐开发顺序

建议按以下顺序逐步派发给 Codex：

1. 基础工程骨架
2. 用户 Web AI 主入口
3. Go 后端 AI 会话与匹配能力
4. 用户 Web 服务与声音承接
5. 管理后台运营骨架
6. 订单、评价、投诉
7. 通知、筛选联动、运营增强
8. 支付与商业化增强

## 3. P0 任务列表

### A. 基础工程骨架

#### P0-01 初始化用户 Web 工程骨架

- 优先级：P0
- 端：用户 Web
- 简述：创建 Vue 工程基础目录、路由、基础布局、公共状态枚举和 API adapter 骨架。

#### P0-02 初始化管理后台工程骨架

- 优先级：P0
- 端：管理后台 Web
- 简述：创建后台项目目录、登录页占位、基础路由、列表页公共布局和 API adapter 骨架。

#### P0-03 初始化 Go 后端工程骨架

- 优先级：P0
- 端：Go 后端
- 简述：建立 `interface/application/domain/infrastructure` 目录、基础 HTTP 服务、配置加载、路由注册和统一响应结构。

#### P0-04 建立数据库迁移基础设施

- 优先级：P0
- 端：Go 后端 / MySQL
- 简述：接入 migration 目录规范，建立基础迁移脚本模板和初始化表结构执行流程。

### B. 用户身份与基础资料

#### P0-05 实现用户登录接口骨架

- 优先级：P0
- 端：Go 后端
- 简述：提供手机号验证码登录和微信登录占位接口，返回统一用户身份结构。

#### P0-06 实现用户基础资料模块

- 优先级：P0
- 端：Go 后端
- 简述：实现 `User`、`UserProfile` 实体、资料查询与保存接口，以及性别一次确认约束。

#### P0-07 实现用户登录与资料补全页面

- 优先级：P0
- 端：用户 Web
- 简述：完成登录页和首次资料补全页，接入登录接口或 mock，并处理资料缺失跳转。

### C. AI 主入口与对话

#### P0-08 实现首页 AI 主入口页面

- 优先级：P0
- 端：用户 Web
- 简述：完成首页核心呼吸球、文本输入入口、语音入口占位、推荐承接区、高级筛选入口和次数提示。

#### P0-09 实现 AI 对话页

- 优先级：P0
- 端：用户 Web
- 简述：完成聊天流、输入区、快捷回复、AI 卡片承接区和安全兜底文案。

#### P0-10 实现 AI 会话领域模块

- 优先级：P0
- 端：Go 后端
- 简述：实现 `AiSession`、`AiMessage`、仓储接口、创建会话、发送消息、查询会话详情接口。

#### P0-11 实现匹配次数限制接口

- 优先级：P0
- 端：Go 后端
- 简述：实现单用户每日匹配次数限制逻辑和剩余次数查询接口。

#### P0-12 实现 AI 推荐结果承接接口

- 优先级：P0
- 端：Go 后端
- 简述：实现匹配请求、匹配结果快照和推荐结果查询接口，AI 结果允许先 mock。

#### P0-13 接通首页与 AI 对话页的 API adapter

- 优先级：P0
- 端：用户 Web
- 简述：将首页和 AI 对话页与会话、消息、匹配次数、推荐结果接口打通。

### D. 服务与声音承接

#### P0-14 实现服务浏览页

- 优先级：P0
- 端：用户 Web
- 简述：完成服务分类、服务方卡片列表、基础筛选区和空态/错误态。

#### P0-15 实现服务方详情页

#### P0-16 实现服务浏览后端查询模块

- 优先级：P0
- 端：Go 后端
- 简述：实现服务分类、服务方列表、服务方详情查询接口和对应查询对象。

#### P0-17 实现声音页基础播放页面

- 优先级：P0
- 端：用户 Web
- 简述：完成声音分类 tabs、播放卡片、推荐列表和基础播放控制 UI。

#### P0-18 实现声音内容后端模块

- 优先级：P0
- 端：Go 后端
- 简述：实现声音分类、声音列表、声音详情、播放记录占位接口。

#### P0-19 实现“我的”页基础页面

- 优先级：P0
- 端：用户 Web
- 简述：完成我的资料、订单入口、设置入口、钱包占位和基础资产展示。

### E. 管理后台运营骨架

#### P0-20 实现后台登录页与鉴权骨架

- 优先级：P0
- 端：管理后台 Web / Go 后端
- 简述：完成后台登录页、登录接口占位、路由守卫和基础权限骨架。

#### P0-21 实现服务方审核列表页

- 优先级：P0
- 端：管理后台 Web
- 简述：完成服务方审核列表、筛选项、状态标签和分页。

#### P0-22 实现服务方审核详情与审核操作

- 优先级：P0
- 端：管理后台 Web / Go 后端
- 简述：完成服务方详情、审核通过、拒绝、补充资料等操作闭环。

#### P0-23 实现服务项目管理模块

- 优先级：P0
- 端：管理后台 Web / Go 后端
- 简述：完成服务项目列表、新增、编辑、启停用能力。

#### P0-24 实现声音内容管理模块

- 优先级：P0
- 端：管理后台 Web / Go 后端
- 简述：完成声音内容列表、新增、编辑、上下架能力。

#### P0-25 实现后台平台配置模块骨架

- 优先级：P0
- 端：管理后台 Web / Go 后端
- 简述：提供基础运营配置项查询和更新能力，为次数限制、首页配置等预留入口。

## 4. P1 任务列表

### A. 订单与状态机

#### P1-01 实现订单实体与状态机

- 优先级：P1
- 端：Go 后端
- 简述：定义 `Order` 实体、状态枚举、状态日志和合法迁移规则。

#### P1-02 实现订单创建接口

- 优先级：P1
- 端：Go 后端
- 简述：支持用户从服务方详情发起订单，生成订单快照和初始状态。

#### P1-03 实现订单列表与详情页

- 优先级：P1
- 端：用户 Web
- 简述：展示用户订单列表、订单详情、状态流转信息和空态/错误态。

#### P1-04 实现后台订单管理模块

- 优先级：P1
- 端：管理后台 Web / Go 后端
- 简述：完成订单列表、订单详情和基础异常处理入口。

### B. 评价与投诉

#### P1-05 实现评价模块

- 优先级：P1
- 端：Go 后端 / 用户 Web
- 简述：支持完单后提交评分与评价内容，并在服务方详情中展示评价摘要。

#### P1-06 实现投诉模块

- 优先级：P1
- 端：Go 后端 / 用户 Web / 管理后台 Web
- 简述：支持用户提交投诉，后台查看投诉列表与处理状态。

### C. 服务方审核流增强

#### P1-07 实现服务方资料审核状态机

- 优先级：P1
- 端：Go 后端
- 简述：集中实现服务方资料审核状态流转、日志记录和驳回原因结构。

- 优先级：P1
- 端：管理后台 Web / 用户 Web / Go 后端
- 简述：让后台审核结果能影响服务方可见性、可接单状态和详情展示。

### D. 筛选与通知

#### P1-09 实现手动筛选与 AI 推荐联动

- 优先级：P1
- 端：用户 Web / Go 后端
- 简述：完成服务场景、风格偏好、价格等筛选参数与 AI 推荐结果联动。

#### P1-10 实现基础通知中心

- 优先级：P1
- 端：用户 Web / Go 后端
- 简述：提供订单、审核、系统消息的基础通知列表与已读逻辑。

- 简述：提供会员权益、购买入口、权益展示和后台配置能力。

#### P2-04 实现广场动态模块

- 优先级：P2
- 端：用户 Web / Go 后端 / 管理后台 Web
- 简述：支持动态发布、浏览、身份标识和内容治理。

## 6. 推荐派工方式

为了让 Codex 更稳定地连续开发，建议每次只派发以下类型之一：

1. 一个前端页面
2. 一个后端领域模块
3. 一个后台列表加详情
4. 一个状态机
5. 一组 migration

不建议直接派发：

- “把 P0 全做完”
- “把用户端都补齐”
- “把订单支付一起做了”
