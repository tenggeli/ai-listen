# Listen 后续 AI 开发任务列表

本文件用于整理 Listen 项目后续可持续交给 Codex 执行的开发任务。

说明：

- 以下列表基于当前代码现状与最新原型更新
- 已完成的任务会标注为 `已完成`
- 后续任务优先围绕现有代码继续推进，不重复初始化工程
- 配置统一优先读取 `~/conf/listenbase.cof`
- 数据相关开发尽量按数据库读取设计
- 所有第三方交互先 mock，当前不做真实对接
- 支付相关开发放在最后，最终支付先 mock 成功

## 1. 当前代码现状摘要

已完成：

- 用户 Web AI 主入口页面
- 用户 Web AI 对话页
- Go 后端 AI 会话与每日匹配次数接口
- Go 后端 AI 推荐结果返回骨架（Mock Match Service）
- 管理后台服务方审核模块
- MySQL 迁移脚本与 `memory/mysql` 仓储切换
- AI 用例基础单测
- 覆盖登录、资料、性格设置、服务、订单、评价投诉链路的完整原型

未完成：

- 用户登录与资料后端
- 性格设置后端
- 服务页、声音页、我的页代码实现
- 订单、评价、投诉后端与前端真实页面
- 管理后台登录鉴权与其他运营模块
- 用户 App 业务页面
- 第三方真实对接

## 7. 推荐派单方式

推荐优先派发这四类任务：

1. “一个原型页 + 一个 API adapter”
2. “一个后端模块 + migration + 单测”
3. “一个订单 / 资料 / 评价状态机”
4. “一个配置改造任务”

不建议再派发：

- “初始化整个工程骨架”
- “把 P0 全部一次完成”
- “先做任何第三方真实对接”
