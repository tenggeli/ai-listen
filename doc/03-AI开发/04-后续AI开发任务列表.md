# Listen 后续 AI 开发任务列表

本文件用于整理 Listen 项目后续可持续交给 Codex 执行的开发任务。

说明：

- 以下列表基于当前代码现状更新
- 已经完成的任务会标注为 `已完成`
- 后续任务优先围绕现有代码继续推进，不重复初始化工程

## 1. 当前代码现状摘要

已完成：

- 用户 Web AI 主入口页面
- 用户 Web AI 对话页
- Go 后端 AI 会话与每日匹配次数接口
- Go 后端 AI 推荐结果返回骨架（Mock Match Service）
- 管理后台服务方审核模块
- MySQL 迁移脚本与 `memory/mysql` 仓储切换
- AI 用例基础单测

未完成：

- 用户登录与资料
- 服务页、声音页、我的页
- 管理后台登录鉴权与其他运营模块
- 用户 App 业务页面
- 订单、评价、投诉、支付

## 2. 当前推荐开发顺序

建议按以下顺序逐步派发给 Codex：

1. 用户 Web 与后端 AI 接口真实联调
2. 用户 Web 服务浏览页
3. 用户 Web 声音页
4. 用户 Web 我的页
5. 管理后台登录鉴权骨架
6. 管理后台服务项目管理
7. 管理后台声音内容管理
8. 订单、评价、投诉骨架
9. 用户 App 页面化
10. 支付与商业化增强

## 7. 推荐派单方式

推荐优先派发这三类任务：

1. “现有模块联调”
2. “一个新页面 + 一个 API adapter”
3. “一个后端查询模块 + migration + 单测”

不建议再派发：

- “初始化整个工程骨架”
- “把 P0 全部一次完成”
- “先做支付或提现”
