# Listen Codex 标准任务模板

本文件用于给 Codex 下达稳定、低歧义、可直接执行的开发任务。

原则：

- 一次只下达一个清晰任务
- 明确端、范围、约束、交付物
- 默认不扩散到支付、结算、提现、VIP
- 默认遵循 `doc/03-AI开发/00-Codex-启动入口.md`、`doc/03-AI开发/01-技术方案设计.md`、`doc/03-AI开发/02-AI-Coding-指导.md`

## 1. 通用模板

```text
请基于以下文档完成开发任务：

- doc/03-AI开发/00-Codex-启动入口.md
- doc/03-AI开发/01-技术方案设计.md
- doc/03-AI开发/02-AI-Coding-指导.md

任务目标：
实现【模块名称】的最小可运行版本。

任务范围：
- 端：用户 Web / 管理后台 / 用户 App / Go 后端
- 页面或模块：【填写具体对象】
- 只做：【填写本次范围】
- 不做：【填写明确排除项】

实现要求：
1. 遵循现有分层与命名规范
2. 业务逻辑不要直接写进页面模板或 controller
3. 如接口未完成，可先 mock，但要保留真实 API 接入位置
4. 状态要覆盖 idle / loading / success / empty / error
5. 代码尽量保持最小闭环，可运行、可扩展

交付要求：
1. 直接修改代码，不只给方案
2. 说明本次改了哪些文件
3. 说明如何验证
4. 若有未完成项，明确列出阻塞点

额外约束：
- 默认不实现支付、结算、提现、VIP
- 不擅自扩大需求范围
- 若发现文档冲突，以 AI 主入口优先、P0 优先为准
```

## 2. 页面开发模板

适用于：首页、AI 对话页、服务页、声音页、后台列表页。

```text
请基于以下文档：

- doc/03-AI开发/00-Codex-启动入口.md
- doc/03-AI开发/01-技术方案设计.md
- doc/03-AI开发/02-AI-Coding-指导.md
- 对应 PRD
- 对应原型 HTML

实现【页面名称】页面的最小可运行版本。

要求：
1. 使用现有技术栈与目录结构
2. 页面拆分为 views / components / domain / application / api
3. 页面必须包含：【列出核心区块】
4. 页面状态包含：idle、loading、success、empty、error
5. 若后端接口未完成，可先 mock，但要保留 adapter 层
6. 不要把复杂业务判断写在模板里
7. 输出后请说明页面入口、关键交互和验证方式

本次不做：
- 【如支付、复杂筛选、真实流式接口等】
```

## 3. 后端模块模板

适用于：AI 会话、订单、服务方审核、声音内容等后端模块。

```text
请基于以下文档：

- doc/03-AI开发/00-Codex-启动入口.md
- doc/03-AI开发/01-技术方案设计.md
- doc/03-AI开发/02-AI-Coding-指导.md

实现 Go 后端【模块名称】的最小闭环。

要求：
1. 按 interface / application / domain / infrastructure 分层
2. 先定义 entity、repository interface、use case、dto、controller
3. 如有状态流转，必须集中到实体方法或领域服务
4. 提供最小可用接口：【列出接口】
5. 需要 migration
6. 需要基础单测
7. 若第三方能力未接入，可先 mock

本次不做：
- 【如真实支付、真实 AI 网关、复杂风控等】
```

## 6. 重构模板

适用于：目录整理、分层拆分、DTO 收敛、状态机收口。

```text
请对【模块名称】进行小范围重构。

目标：
- 提升可维护性
- 符合当前 AI Coding 分层规范

要求：
1. 不改变现有对外行为
2. 只重构【指定范围】
3. 将散落逻辑收敛到 domain / application / api adapter
4. 不要扩大成全项目重构
5. 输出时说明重构前问题、重构后结构、验证结果
```

## 7. 推荐填写方式

为了让 Codex 更稳定，建议每次任务至少写清楚这 6 个字段：

1. 做什么模块
2. 属于哪个端
3. 本次只做什么
4. 本次明确不做什么
5. 是否允许 mock
6. 是否需要测试 / migration / 接口

## 8. 一个可直接复制的示例

```text
请基于以下文档完成开发任务：

- doc/03-AI开发/00-Codex-启动入口.md
- doc/03-AI开发/01-技术方案设计.md
- doc/03-AI开发/02-AI-Coding-指导.md
- doc/02-PRD/用户端PRD.md
- doc/04-原型素材/listen_home_v2_dark.html

任务目标：
实现用户 Web 首页 AI 主入口的最小可运行版本。

任务范围：
- 端：用户 Web
- 页面：首页
- 只做：核心呼吸球、文本输入区、推荐承接区、匹配次数提示、高级筛选入口
- 不做：真实支付、完整订单、复杂推荐算法、真实语音输入

实现要求：
1. 使用 Vue
2. 按 views / components / domain / application / api 分层
3. 页面状态包含 idle、loading、success、error、limited
4. 推荐结果可先使用 mock 数据
5. 后续可无缝替换为真实 API

交付要求：
1. 直接完成代码修改
2. 说明改动文件
3. 说明启动与验证方式
4. 标出后续待接入项
```
